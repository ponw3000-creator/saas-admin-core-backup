# 高危操作与表单防呆
1. **权限移交**：任何不可逆的超管移交操作，必须伴随 Loading 延迟（至少 1.5s 的摩擦感），且成功后必须执行 `localStorage.clear()` 并强制定向至 `/login`。
2. **数值边界锁死**：所有配置时长的数字输入框（如会话超时、闲置登出），必须配置物理 `min` 和 `max` 属性，并在 UI 气泡中明示边界数值。
3. **接口防暴击**：登录、注册、保存等核心提交按钮，必须配备 `loading` 状态与 `throttle/debounce` 节流控制（防并发）。

## 租户注册风控 (Tenant Onboarding Security)
- **唯一性阻断**：注册表单中的"企业名称"必须配置异步防重复校验（Async Validator）。若探测到重复，需抛出红色错误提示，并引导用户切换至登录模式。

## 凭证生命周期管理 (Credential Lifecycle)
- **零信任初始凭证**：系统生成的任何初始密码/邀请码，必须附加有效期（通常为 48 小时）。
- **首次登录强制改密**：任何使用初始随机密码登录的账号，登录成功后必须强制进入"修改密码"流程，严禁直接进入业务系统。
